The XOR function as defined by the following truth table is a classic example of a non-linear function that cannot be approximated with just linear functions, which is why non-linear activation functions and multiple layers are required in neural networks.
The following is a simple 2-2-1 neural network with the ReLU (rectified linear unit) activation function and a output function that outputs 0 or 1.
Try to adjust the weights manually so this network manages to compute the XOR function!

$$\text{Inputs: }\vec{v_1} = \begin{bmatrix}1\\0\end{bmatrix}, W_1 = \begin{bmatrix}0.5 && 0.5 \\ 0.5 && 0.5 \end{bmatrix}, \vec{b_1} = \begin{bmatrix}0\\0\end{bmatrix}, W_2 = \begin{bmatrix}0.8 && 0.4 \end{bmatrix}, \vec{b_2} = \begin{bmatrix}-0.3\end{bmatrix}$$

$$\text{Computed Values: }\vec{v_2} = W_1\vec{v_1}+\vec{b_1} = \begin{bmatrix}0.5\\0.5\end{bmatrix}, \vec{v_3} = f(\vec{v_2}) = \begin{bmatrix}0.5\\0.5\end{bmatrix}$$

$$\vec{v_4} = W_2\vec{v_3}+\vec{b_2} = \begin{bmatrix}0.3\end{bmatrix}, \vec{v_5} = g(\vec{v_4}) = \begin{bmatrix}0\end{bmatrix}$$

$$\text{Activation Function: } f(x) = \begin{cases} x \text{ if } x \ge 0 \\ 0 \text { if } x \lt 0 \end{cases} \text{ , Output Function: } g(x) = \begin{cases} 1 \text{ if } x \ge 0.5 \\ 0 \text{ if } x \lt 0.5 \end{cases}$$

Notes: This demonstration is optimized for computers. While it will still function on mobile devices and has a good amount of responsiveness coded in, the weights in the graph will not display properly on mobile.
Note that all calculations are rounded to 2 decimal places.
